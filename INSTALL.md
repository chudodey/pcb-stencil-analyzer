# Установка и запуск Stencil Analyzer

## Системные требования

- **ОС**: Windows 10/11 (64-bit)
- **Python**: 3.8 или новее
- **Память**: 4 ГБ ОЗУ минимум
- **Диск**: 500 МБ свободного места

## Установка

1. **Установите Python** с [python.org](https://python.org)
   - Отметьте "Add Python to PATH" при установке
2. **Скачайте проект** в папку `v13_final`

3. **Установите зависимости**:

   ```cmd
   cd v13_final
   pip install -r requirements.txt
   ```

4. **Проверьте установку**:
   ```cmd
   python main.py --version
   ```

## Запуск

### Обычный режим:

```cmd
python main.py
```

### Режим отладки (рекомендуется):

```cmd
run.bat
```

## Настройка папок

Создайте структуру:

```
v13_final/
├── test_gerbers/    # Gerber-файлы (*.gbr)
├── test_scan/       # Сканы трафаретов (*.jpg, *.png)
└── test_results/    # Результаты (создается автоматически)
```

   - ФИО оператора

## Мониторинг процесса

### Логи:

- `test_results/application.log` - общий лог приложения
- `test_results/{заказ}/` - логи конкретного заказа

### Результаты:

- `{номер}_1_gerber.png` - эталон из Gerber
- `{номер}_2_scan.png` - исходный скан
- `{номер}_4_compared.png` - визуальное сравнение
- `{номер}_6_log_short.txt` - краткий отчет

## Решение проблем

### Файлы не находятся:

- Проверьте номер заказа в имени файла
- Убедитесь, что файлы в правильных папках
- Проверьте расширения (.gbr, .jpg, .png)

### Плохое совмещение:

- Используйте разрешение 1200 DPI при сканировании
- Проверьте освещение и чистоту трафарета
- Убедитесь в правильной ориентации

### Ошибки при запуске:

```cmd
# Переустановите зависимости
pip install -r requirements.txt

# Проверьте Python
python --version
```

## Техническое обслуживание

### Очистка результатов:

```cmd
python clean.py
```

### Обновление конфигурации:

Отредактируйте `config.ini` для изменения:

- Путей к папкам
- Параметров обработки
- Порогов качества

### Резервное копирование:

Регулярно сохраняйте:

- Папку `test_results/`
- Файл `config.ini`
